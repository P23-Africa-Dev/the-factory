# Health API

## Feature Overview

The Health API provides a lightweight status endpoint for connectivity checks, deployment verification, uptime monitoring, and frontend environment validation.

Frontend teams can use this endpoint to:

1. Confirm the backend is reachable.
2. Validate environment wiring during local development.
3. Support infrastructure or uptime checks.

## API Endpoints

### Get API Health

- URL: `/api/v1/health`
- Method: `GET`

## Authentication Requirements

- Authentication: Not required
- Authorization: Public endpoint

## Request Structure

### Headers

| Header | Required | Value |
| --- | --- | --- |
| `Accept` | Recommended | `application/json` |

### Path Parameters

- None

### Query Parameters

- None

### Request Body

- None

## Response Structure

### Success Response

Status: `200 OK`

```json
{
  "success": true,
  "message": "API is healthy",
  "data": {
    "service": "Factory API",
    "environment": "local",
    "timestamp": "2026-04-03T12:00:00+00:00"
  },
  "errors": null
}
```

### Error Response

This endpoint does not currently define a feature-specific error payload. Platform-level failures may still occur.

Example infrastructure/server failure:

Status: `500 Internal Server Error`

```json
{
  "success": false,
  "message": "Server error",
  "data": null,
  "errors": {
    "server": [
      "An unexpected error occurred."
    ]
  }
}
```

## Validation Rules

- No request validation rules apply.

## Status Codes And Error Messages

| Status Code | Meaning | Notes |
| --- | --- | --- |
| `200` | API reachable and healthy | Expected success response |
| `500` | Unexpected server failure | Infrastructure or application-level issue |

## Example Requests

### cURL

```bash
curl --request GET \
  --url http://localhost:8080/api/v1/health \
  --header 'Accept: application/json'
```

### JavaScript Fetch

```javascript
const response = await fetch('http://localhost:8080/api/v1/health', {
  headers: {
    Accept: 'application/json',
  },
});

const payload = await response.json();
```

## Frontend Integration Notes

1. Treat this endpoint as a liveness check, not a full readiness guarantee for domain services.
2. `data.environment` is useful for debugging local, staging, and production mismatches.
3. `data.timestamp` is generated by the backend and can be used to verify server-side response freshness.
